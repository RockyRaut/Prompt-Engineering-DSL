# File Explanations

This document explains the purpose and key code elements of every important file in this project.

## Root Files

### `README.md`
- Provides a short project summary.
- Lists installation commands and usage examples:
  - `python -m venv .venv`
  - `.venv\Scripts\pip install -r requirements.txt`
  - `streamlit run prompt_dsl/ui/streamlit_app.py`
  - `python -m prompt_dsl.main samples/example.dsl`

### `requirements.txt`
- Lists Python dependencies required to run the project.
- Current dependencies:
  - `ply` — used for lexing and parsing the DSL.
  - `streamlit` — optional UI frontend.
  - `pytest` — test runner.

### `samples/example.dsl`
- Example Prompt DSL source configured to summarize article.txt.
- Defines an ArticleSummarizer agent with the Summarizer role.
- Demonstrates all supported syntax: agent, role, task, input, constraints, and output format.
- Reads from the project root file `article.txt` and generates a summarization prompt.
- Serves as the default example in both the CLI and Streamlit UI.

### `tests/test_parser.py`
- Contains automated tests for the DSL parser and prompt generator.
- Uses:
  - `Lexer.tokenize(dsl)` to lex input text.
  - `Parser(dsl).parse()` to parse the source into an AST.
  - `PromptGenerator.generate(program)` to build the final prompt text.
- Includes regression tests:
  - `test_example_dsl` — verifies basic waste classification example still works.
  - `test_parser_accepts_missing_final_newline` — handles missing final newline.
  - `test_summarization_example_generates_a_helpful_summary_prompt` — validates the new article summarization DSL generates appropriate prompt text.

## AST Module

### `prompt_dsl/ast/nodes.py`
- Defines the AST node classes used by the parser:
  - `AgentNode`
    - Fields: `name`, `role`, `tasks`, `inputs`, `constraints`, `output`, `memory`, `rag`, `tools`
  - `ProgramNode`
    - Field: `agents`
- `ProgramNode.to_dict()` converts the AST into a normalized dictionary representation.
- This module provides the structured program model used throughout the pipeline.

## Lexer Module

### `prompt_dsl/lexer/lexer.py`
- Implements the DSL lexer using `PLY`.
- Defines token names in `tokens`:
  - `AGENT`, `ROLE`, `TASK`, `INPUT`, `OUTPUT`, `CONSTRAINT`, `MEMORY`, `RAG`, `TOOL`, `VALUE`, `NEWLINE`
- `Token` dataclass holds token metadata:
  - `type`, `value`, `line`
- PLY token rules:
  - `t_newline` detects newlines and increments line numbers.
  - `t_COMMENT` ignores comment lines starting with `#`.
  - `t_KEYWORD` matches DSL keywords and returns them as token types.
  - `t_VALUE` matches the remainder of a line after a keyword or any non-keyword text.
  - `t_error` reports illegal characters.
- `build_lexer()` returns a configured PLY lexer instance.
- `Lexer.tokenize(text)` runs the lexer and returns a list of `Token` objects.

## Parser Module

### `prompt_dsl/parser/parser.py`
- Implements the DSL parser using `PLY`.
- Defines `ParseError` for syntax or parse failure.
- Grammar rules include:
  - `program : statements`
  - `statements : statements statement | statement`
  - `statement : AGENT optional_value NEWLINE | ROLE optional_value NEWLINE | ...`
  - `statement : VALUE NEWLINE | VALUE`
  - `statement : NEWLINE`
  - `optional_value : VALUE |` (empty)
- Uses the PLY parser generated by `yacc.yacc()`.
- `Parser(source)` stores raw DSL text.
- `Parser.parse()` runs the parser and translates the parsed rules into an AST.
- `_build_ast(statements)` converts the parser output into a `ProgramNode`:
  - Creates or selects a current `AgentNode`.
  - Maps keyword statements into agent fields.
  - Uses `VALUE` statements as fallback tasks or inputs.

### `prompt_dsl/parser/parser.out`
- Generated by PLY when the parser is built.
- Contains the grammar rules and parser states.
- Useful for debugging or understanding the generated parser.

### `prompt_dsl/parser/parsetab.py`
- Generated PLY parser table cache.
- Stores parsing tables so the parser can load them faster.
- It is not handwritten source code; PLY regenerates it as needed.

## Semantic Validator

### `prompt_dsl/semantic/validator.py`
- Defines semantic validation rules for parsed programs.
- `SemanticError` class carries an error message and optional line number.
- `Validator.validate(program)` returns a list of semantic errors.
- Current checks:
  - At least one `AGENT` declaration is present.
  - Only one `AGENT` is allowed.
  - `ROLE` is required.
  - `TASK` is required.
  - `OUTPUT`, if present, must be one of: `TEXT`, `JSON`, `MARKDOWN`, `XML`.

## Prompt Generator

### `prompt_dsl/generator/prompt_generator.py`
- Converts a validated AST into a final prompt string.
- Contains templates for roles and constraints:
  - `ROLE_TEMPLATES` maps role values to friendly descriptions (includes `SUMMARIZER: "You are a helpful summarizer."`).
  - `CONSTRAINT_TEMPLATES` maps constraint values to readable instructions.
- `PromptGenerator.generate(program)` builds prompt sections:
  - role description
  - task
  - input
  - constraints
  - memory flag
  - tool guidance
  - RAG guidance
  - output format instruction
- Joins the prompt segments into a human-readable prompt.

## CLI and Example Runner

### `prompt_dsl/main.py`
- CLI entry point for the DSL pipeline.
- Reads the DSL file path from command-line arguments.
- Execution flow:
  1. Read DSL text from file.
  2. Tokenize with `Lexer.tokenize()`.
  3. Parse with `Parser(source).parse()`.
  4. Validate with `Validator.validate(program)`.
  5. Generate prompt with `PromptGenerator.generate(program)`.
  6. Print the final prompt.
- Returns `0` for success, `1` on validation failure, and `2` for usage errors.

### `prompt_dsl/run_example.py`
- Example script that demonstrates the pipeline in action.
- Prints:
  - source DSL text
  - tokens produced by the lexer
  - AST dictionary from `program.to_dict()`
  - validation results
  - generated prompt
- Useful for manual experimentation and debugging.

## Streamlit UI

### `prompt_dsl/ui/streamlit_app.py`
- Streamlit application for interactive DSL editing.
- Title: "Prompt DSL — Article Summarizer"
- Pre-populated with the article summarization example by default.
- Shows a text area with DSL content.
- On button press, it:
  - tokenizes the DSL
  - parses it into an AST
  - validates the program
  - generates a prompt
- Displays the results in the UI:
  - token list
  - AST JSON
  - validation messages
  - generated prompt

### `article.txt`
- Sample input file in the project root containing text to be summarized.
- Referenced by the example DSL as the input source.
- Used to demonstrate end-to-end DSL workflow from input specification to generated prompt.

## Notes

- `prompt_dsl/providers` has been removed from the project.
- The project is explicitly designed to be self-contained and run without external AI provider dependencies.
- The project is now centered around article summarization as its primary use case.
- Generated parser files (`parsetab.py` and `parser.out`) are artifacts produced by PLY and should not be edited by hand.
